Install Vocola

Install Vocola New users should use the following instructions to install Vocola. (Vocola 1.1 users should use these instructions.)
  1. Install Dragon NaturallySpeaking, version 3.5 or later, any edition.

    2.  
  2. Install Python. Version 2.2 is recommended, and version 2.0 is supported, but version 1.5 is not supported.
     
  3. Install NatLink. For Python 2.2 use this version. Follow Joel Gould's instructions. NatLink version 007 or greater is recommended, though Vocola has worked since version 004.
     
  4. Download the Vocola zip file.

    5.  
  5. Extract the Vocola zip file into the NatLink directory. For example, if you installed NatLink in C:\NatLink, you should choose C:\NatLink as the folder to extract to. After unpacking you should see folder C:\NatLink\Vocola, with subfolders Commands, Doc, Exec, Samples, and Simpscrp.
     
  6. From the Vocola\Exec folder, copy the files _vocola_main.py and VocolaUtils.py to the NatLink\MacroSystem folder. This activates Vocola's built-in voice commands.

    7.  
  7. Associate the .vcl file extension (used for Vocola command files) with your favorite text editor. This allows you to open your Vocola files using built-in voice commands. An easy way to set up this association is to right-click on any .vcl file (such as Vocola\Samples\notepad.vcl), select "Open", and pick your favorite text editor (for example, Notepad).

    8. Emacs users should install gnuserv and associate the .vcl file extension with gnuclientw.exe. (Make sure the directory containing gnuclientw.exe is included in your PATH environment variable.)

  8. Test Vocola. 
    • Copy the sample command file for Notepad (Vocola\Samples\notepad.vcl) to the Vocola\Commands folder. 
    • Say "Start Notepad" to launch Notepad.
    • Say "Test Simple Command". The phrase "It works!" should appear.
    • Say "Edit Voice Commands". Vocola\Commands\notepad.vcl should be opened using the editor you picked in step 7.
    • Change the "Test Simple Command" command to type "It really works!"
    • Save the file, say "Switch to Notepad", and say "Test Simple Command". "It really works!" should appear.

Upgrading from Vocola 1.1

Vocola-generated Python files are named differently in Vocola 2.0 than they were in Vocola 1.1. (For example, outlook.vcl used to generate outlook.py but now generates outlook_vcl.py.) This means Vocola is unlikely to clobber hand-written NatLink files. Vocola 1.1 users should upgrade as follows:

  1. Rename your existing NatLink/Vocola folder
  2. Install Vocola 2.0 as described above
  3. Copy .vcl files from your 1.1 Vocola/Commands folder to your 2.0 Vocola/Commands folder
  4. Delete all Vocola-created .py files in the NatLink/MacroSystem folder (they will all have the same timestamp)
  5. Delete all .pyc files in the NatLink/MacroSystem folder
  6. Restart NatSpeak
